Detailed Description

This is the main class that handles the call flow between module. You will need to inherit from this class and override loadModules to specify what modules you want to load.

The typical intializations step are the following

  • init Initializes ModFlow by providing the custom classes derived from NlSources and NlSinks. (see init for more details)
  • sources() -> declareSource (...) to declare channels which the sources will emit data to
  • sinks() ->declareSink (...) to declare channels that will be connected to external slots from sinks
  • finalize() will configure each loaded module by calling NlModule::initParams, if any, and NlModule::setupNetwork, that each module shall override.

After the intialization phase, externally the sources can be triggered by calling sources() -> callSource, which will emit an event on the specified channel Sinks will connect regular channel events to so called sink channels, hold by a NlSinks -derived class, as specified via NlSinks::connectToSink in the overridden NlSinks::setupNetwork. As an event an a sink-channel is emitted, the callback parentSlot defined in the declaration NlSinks::declareSink is called forwarding the regular channel data to the parent object slot.

Note
Do not call this method directly. Depending on the channel type:
  • If the channel is regular, slot must be bound to the module object owning the channel. Achieve this by calling NlModule::requestConnection from the module.
  • If the channel is sink, it is bound to the external calling object. Achieve this by calling NlSinks::connectToSink
